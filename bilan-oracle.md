# Bilan Lab Git — L'Oracle

## 3 choses apprises dans les Labs Git

1. **Les branches** permettent de travailler en parallèle sans casser le code
   des autres. Chaque fonctionnalité = une branche séparée.

2. **Les Pull Requests** sont un mécanisme de vérification collective :
   on ne merge pas seul, l'Architecte review avant d'intégrer.

3. **Les conflits Git** ne sont pas des erreurs mais un signal normal :
   quand deux personnes modifient la même ligne, Git demande à l'équipe
   de décider quelle version garder.
